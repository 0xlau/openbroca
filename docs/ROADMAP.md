# Roadmap

OpenBroca is pre-1.0. This roadmap describes direction rather than a release promise.

## Current Focus

- Reliable desktop voice capture and final text delivery.
- Provider configuration, authentication, and health checks.
- Local ASR model management.
- Strong onboarding for permissions, shortcuts, and providers.
- Typed, testable renderer-to-main APIs.

## Near Term

- Stabilize provider runtime isolation.
- Improve cross-platform permission detection.
- Stabilize the full desktop test suite and Electron mocks.
- Expand ASR and LLM provider coverage.
- Add clearer release and update flows.
- Improve first-run documentation and troubleshooting.

## Later

- Plugin-style provider distribution.
- Workflow automation beyond plain text insertion.
- More privacy-first local processing paths.
- Richer voice history search and editing.
- Public extension points for community integrations.

## Non-Goals For Now

- Hosting a cloud backend as a required dependency.
- Committing to a stable provider SDK before the desktop workflows settle.
- Shipping private keys, bundled credentials, or model artifacts in the repository.
